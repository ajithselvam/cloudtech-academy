1. What is Jamf Pro?

Jamf Pro is a comprehensive Enterprise Mobility Management (EMM) solution specifically designed for managing Apple devices within organizations. It allows IT administrators to deploy, secure, and manage Mac computers, iPhones, iPads, and Apple TV devices, from initial setup through their entire lifecycle.

Jamf Pro provides a unified platform to automate device enrollment, configure settings, deploy software, enforce security policies, manage inventory, and provide a self-service app store for users.

Key Benefits:

  • Streamlined Deployment: Automate device setup from out-of-the-box.
  • Centralized Management: Manage thousands of devices from a single console.
  • Enhanced Security: Enforce encryption, password policies, and deploy security updates.
  • Application Deployment: Distribute and manage apps effortlessly.
  • User Empowerment: Provide users with self-service options for apps and resources.
  • Inventory & Reporting: Gain real-time insights into your device fleet.

2. Key Concepts & Architecture

Understanding the core components and concepts is crucial for effective Jamf Pro administration.

A. Jamf Pro Server:

  • The central hub for all management tasks. It can be hosted on-premise or in the cloud (Jamf Cloud).
  • Consists of a web application, a database (MySQL/MariaDB), and a Tomcat server.

B. Managed Apple Devices:

  • Mac computers (macOS)
  • iPhones and iPads (iOS/iPadOS)
  • Apple TV (tvOS)

C. Jamf Agent (for macOS):

  • A lightweight client-side application installed on macOS devices during enrollment.
  • Communicates with the Jamf Pro server to receive commands, send inventory updates, and execute policies.

D. Self Service:

  • A customizable app store-like application for end-users on managed devices.
  • Allows users to install approved applications, run scripts, and access IT resources without contacting support.

E. Apple Push Notification service (APNs):

  • Essential for real-time communication between the Jamf Pro server and managed devices (especially iOS/iPadOS/tvOS, and for certain macOS commands).
  • Enables commands like lock, wipe, install app, update OS, and check in.

F. Device Enrollment Program (DEP) / Automated Device Enrollment (ADE) & Apple School Manager (ASM) / Apple Business Manager (ABM):

  • Apple's programs for zero-touch deployment. Devices purchased directly from Apple or authorized resellers can be automatically enrolled in Jamf Pro during initial setup.

3. Initial Setup & Enrollment

Before managing devices, you need to set up your Jamf Pro instance and enroll devices.

A. Jamf Cloud Instance Access:

  • If using Jamf Cloud, you'll receive a URL (e.g., `https://yourdomain.jamfcloud.com`).
  • Log in with your administrator credentials.

B. Basic Configuration (Initial Login):

  • Create an APNs Certificate: Crucial for device communication. Jamf Pro guides you through this process (requires an Apple ID).
  • Connect to Apple Push Notification service: Ensure it's active.
  • Set up Scope & Restrictions (Initial): Define basic security settings.

C. Device Enrollment Methods:

How devices get managed by Jamf Pro.

  • Automated Device Enrollment (ADE / DEP): For institutionally owned devices purchased through Apple Business/School Manager.
    • Devices are automatically assigned to Jamf Pro when activated.
    • Offers supervised mode for full control.
    # Configuration Steps in Jamf Pro (GUI):
1.  Settings (gear icon) > Global Management > Automated Device Enrollment.
2.  Click "New" and follow prompts to connect to your Apple Business/School Manager account.
3.  Define a "PreStage Enrollment" for device groups (e.g., "Corporate Macs").
4.  Configure setup assistant options (e.g., skip Apple ID, Touch ID setup).
  • User-Initiated Enrollment: For personally owned devices (BYOD) or devices not in ADE/DEP. Users manually enroll.
    • Users browse to your Jamf Pro enrollment URL (e.g., `https://yourdomain.jamfcloud.com/enroll`).
    • They follow prompts to install a profile/agent.
    • Can be configured for user authentication and different enrollment experiences.
    # Configuration Steps in Jamf Pro (GUI):
1.  Settings (gear icon) > Computer Management > Enrollment Methods > User-Initiated Enrollment.
2.  Configure options like authentication, categories, and custom messages.
3.  Direct users to the enrollment URL.
  • Jamf Imaging (macOS): For traditional imaging workflows (less common with modern macOS).

4. Inventory and Reporting

Jamf Pro automatically collects detailed information about all enrolled devices, providing a powerful inventory system.

  • Collected Data: Hardware specifications (model, serial number, CPU, RAM), software installed (applications, versions), user accounts, network configuration, security status (FileVault, OS version).
  • Viewing Inventory:
    • Navigate to Computers or Mobile Devices tabs.
    • Click on any device to view its detailed inventory record.
  • Custom Inventory Attributes: Define custom fields to collect unique data relevant to your organization. 
    # Steps to create a Custom Extension Attribute (macOS example):
1.  Settings (gear icon) > Computer Management > Extension Attributes.
2.  Click "New" > Choose "Script" input type.
3.  Provide a display name (e.g., "Custom App Version").
4.  Paste a shell script that outputs the desired value to stdout.
    Example Script (to get a custom app's version):
    <pre><code class="language-bash">#!/bin/bash
if [ -f "/Applications/MyCustomApp.app/Contents/Info.plist" ]; then
    version=$(defaults read "/Applications/MyCustomApp.app/Contents/Info.plist" CFBundleShortVersionString)
    echo "<result>$version</result>"
else
    echo "<result>Not Installed</result>"
fi</code></pre>
5.  Save the Extension Attribute. It will be collected during the next inventory update.
  • Reporting: Generate detailed reports based on inventory data.
    • Go to Computers > Reports or Mobile Devices > Reports.
    • Create new reports based on specific criteria (e.g., "All Macs with macOS Ventura," "iPads with less than 10% battery").

5. Policies (Software Deployment, Scripts, Settings)

Policies are the primary method for performing automated tasks on managed computers. They can deploy software, run scripts, apply settings, and more.

A. Creating a Policy:

# Navigation: Computers > Policies > New (+)

# Key sections when creating a policy:
1.  General:
    -   Display Name (e.g., "Install Microsoft Office")
    -   Category (e.g., "Software Deployment")
    -   Trigger:
        -   **Recurring Check-in:** Runs every 15 minutes, once a day, etc.
        -   **Enrollment Complete:** Runs immediately after a device enrolls.
        -   **Login/Logout:** Runs when a user logs in/out.
        -   **Network State Change:** Runs when network status changes.
        -   **Custom Trigger:** Triggered by `sudo jamf policy -event <custom_trigger_name>` (useful for scripts).
    -   Execution Frequency: Once per computer, Once per user, Every time, Once every X days/weeks/months.

2.  Packages: Add pre-uploaded software packages (.pkg, .dmg).
    -   Action: Install.
    -   Options: Restart required, fill user templates.

3.  Scripts: Add custom shell scripts to run on devices.
    -   Priority: Before/After/At Reboot.
    -   Parameters: Pass variables to your scripts.
    Example Script (in Jamf Policy):
    <pre><code class="language-bash">#!/bin/bash
# Parameter 4: Custom message from Jamf Pro policy
MESSAGE="$4" 
echo "Policy triggered at $(date) with message: $MESSAGE" >> /Library/Application Support/Jamf/Logs/policy_log.log
/usr/bin/osascript -e 'display notification "Hello from Jamf Policy!" with title "Jamf Alert"'
</code></pre>

4.  Printers: Add/remove printers.
5.  Disk Encryption: Manage FileVault.
6.  Maintenance: Perform disk repairs, update inventory.
7.  User Interaction: Display messages or restart prompts.
8.  Files and Processes: Manage files, run commands, control processes.
9.  Restrictions: Enforce various macOS restrictions (e.g., disable camera).

B. Deploying Software (Example: Deploy Google Chrome)

  1. Package Google Chrome: Download the `.dmg` or `.pkg` installer from Google. Use Jamf Composer (macOS utility) to convert it to a `.pkg` if necessary, or simply upload the `.dmg` if Jamf supports its direct deployment.
  2. Upload Package to Jamf Distribution Point: Jamf Pro stores packages on a distribution point (cloud or on-premise). 
    # Navigation: Settings (gear) > Server Infrastructure > Distribution Points > Your DP > Packages > Upload.
  3. Create a Policy: 
    # Navigation: Computers > Policies > New (+)
-   Display Name: Install Google Chrome
-   Category: Software Deployment
-   Trigger: Recurring Check-in (e.g., every day)
-   Execution Frequency: Once per computer
-   Packages: Add the uploaded "Google Chrome.pkg"
-   Scope: (See "Scoping" section) Add your "All Managed Macs" Smart Group.
-   Self Service: (Optional) Make it available in Self Service.
  4. Test: Enroll a test device and verify Chrome is installed.

6. Configuration Profiles (Settings Management)

Configuration Profiles are XML files (or payloads) that enforce settings on macOS, iOS, iPadOS, and tvOS devices. They are ideal for persistent, required settings.

A. Creating a Configuration Profile:

# Navigation: Computers > Configuration Profiles > New (+) (for macOS)
# OR Mobile Devices > Configuration Profiles > New (+) (for iOS/iPadOS)

# Key payloads (settings) when creating a profile:
1.  General:
    -   Name (e.g., "Wi-Fi Settings - Corporate Network")
    -   Category
    -   Level: User Level (applies to user session) or Computer/Device Level (applies to device itself, even without user login).

2.  Wi-Fi: Configure Wi-Fi networks (SSID, security type, password).
3.  Restrictions: Disable cameras, force passcode, restrict app installation.
4.  Passcode: Enforce passcode requirements (length, complexity, age).
5.  Web Clips: Create shortcuts to websites on the home screen.
6.  Email: Configure email accounts.
7.  Certificates: Deploy certificates for network access or VPN.
8.  Privacy Preferences Policy Control (PPPC): Grant permissions to apps (e.g., screen recording, full disk access).
9.  Custom Settings: Deploy custom plist files.

B. Deploying a Wi-Fi Profile (Example: iOS/iPadOS)

  1. Create a Configuration Profile: 
    # Navigation: Mobile Devices > Configuration Profiles > New (+)
-   Name: Corporate Wi-Fi
-   Category: Network
-   Level: Device Level (applies to the device)
-   Payloads: Add "Wi-Fi" payload
    -   SSID: Your_Corporate_WiFi_SSID
    -   Security Type: WPA/WPA2 Enterprise (or appropriate)
    -   Protocol: EAP-TLS (or appropriate, often requires certificates)
    -   Auto Join: Checked
  2. Scope: Apply to your "All Managed iPads" Smart Group.
  3. Test: Verify test iPads automatically connect to the corporate Wi-Fi.

7. Patch Management

Keeping macOS and third-party software updated is critical for security and stability.

A. macOS Updates:

  • Jamf Pro can manage macOS major upgrades and minor updates.
  • Use policies to defer updates, enforce updates, or make them available via Self Service.
    • # Example Policy for macOS Update:
# Navigation: Computers > Policies > New (+)
-   Display Name: Upgrade to macOS Sonoma
-   Category: OS Updates
-   Trigger: Manual (User-initiated via Self Service) or Recurring Check-in (for forced updates)
-   Execution Frequency: Once per computer
-   Payloads: Add "Operating System"
    -   Action: Install an update
    -   macOS Version: Latest (or specific version)
    -   Options: Restart if needed, deferrals, force installs.
-   Scope: Smart Group of devices eligible for Sonoma upgrade.
-   Self Service: Make available (with icon and description).

B. Third-Party App Patching:

  • Jamf Pro 10.27+ introduced **App Installers** for streamlined third-party app management and patching (e.g., Chrome, Firefox, Zoom). This leverages Jamf's Content Distribution Network (CDN).
  • For other apps, you might need custom scripts or tools like **Jamf Patch Management (for custom titles)**, or integrations with tools like AutoPkg.
    • # Example using App Installers (Jamf Pro GUI):
1.  Computers > App Installers.
2.  Click "New" or browse available titles.
3.  Select an app (e.g., Google Chrome).
4.  Configure:
    -   Deployment Type: "Install automatically" (for forced updates) or "Make available in Self Service".
    -   Scope: Apply to relevant Smart Groups.
5.  Jamf Pro will automatically download, deploy, and update the app when new versions are available.

8. Application Management

Deploying and managing applications is a core function.

  • Packaging Applications:
    • `.pkg` files are the preferred format for macOS software deployment.
    • Jamf Composer (included with Jamf Pro) helps create `.pkg` installers from `.dmg` files or by snapshotting installations.
  • App Store Apps (iOS/iPadOS/tvOS and macOS):
    • Integrate Jamf Pro with your **Apple Business Manager (ABM)** or **Apple School Manager (ASM)** account for managed app distribution.
    • Purchase apps through ABM/ASM for Volume Purchase Program (VPP) and assign them to devices or users in Jamf Pro.
    # Steps for VPP App Deployment:
1.  Settings (gear) > Global Management > VPP Accounts > Add your VPP token.
2.  Apps > New (+) > App Store app or VPP App.
3.  Search for the app, select it.
4.  Configure:
    -   Distribution Method: "Install Automatically" or "Make Available in Self Service".
    -   App Configuration (Managed App Config): Pass specific settings to the app.
    -   Scope: Assign to devices/users.
    -   VPP Assignment: Choose "Assign to device" or "Assign to user".
  • Custom Applications (macOS): For in-house or non-App Store macOS apps, package them as `.pkg` files and deploy via Policies.

9. Self Service

Jamf Self Service empowers end-users to install applications, run scripts, and access resources without requiring direct IT intervention.

  • Making Items Available:
    • When creating a Policy or App/Profile, enable the "Self Service" tab.
    • Configure an icon, description, category, and featured status.
  • Customizing Self Service: 
    # Navigation: Settings (gear) > Self Service (for Computers or Mobile Devices)
-   Branding: Customize icons, banner, colors.
-   Layout: Organize categories, featured items.
-   Notifications: Configure push notifications for new content.
  • Use Cases:
    • Optional software installs (e.g., specialized design software).
    • Running maintenance scripts (e.g., "Clear Print Queue").
    • Accessing documentation or internal web links.
    • Initiating a FileVault encryption.

10. Smart Groups & Static Groups

Groups are fundamental for organizing devices and users, and for scoping policies and profiles.

  • Static Groups:
    • Manually populated groups. You explicitly add or remove devices/users.
    • Useful for small, specific collections (e.g., "Executive Team Macs", "Pilot Program iPads").
    # Navigation: Computers > Static Computer Groups > New (+)
# Or Mobile Devices > Static Mobile Device Groups > New (+)
  • Smart Groups:
    • Dynamically populated groups based on criteria from device inventory.
    • Automatically update as device inventory changes.
    • Essential for automation (e.g., "All Macs with low disk space", "iPads without passcode enforced").
    # Navigation: Computers > Smart Computer Groups > New (+)
# Or Mobile Devices > Smart Mobile Device Groups > New (+)

# Example Smart Group Criteria (for "Macs with macOS 10.15 Catalina"):
-   Criteria: Operating System Version
-   Operator: is
-   Value: 10.15.x

# Example Smart Group for "Devices with App X Installed":
-   Criteria: Application Title
-   Operator: is
-   Value: Microsoft Word.app

# Example Smart Group for "Devices without FileVault Enabled":
-   Criteria: FileVault 2 Status
-   Operator: is not
-   Value: Encrypted

11. Scoping

Scoping defines which devices or users a policy, profile, or application applies to. It's done by assigning to groups.

  • General Principle:
    • You include specific **groups** (Smart or Static) or **individual devices/users**.
    • You can also add **exclusions** (specific groups/devices/users that should NOT receive the item).
  • Location within Jamf Pro: Every policy, configuration profile, and app distribution has a "Scope" tab where you configure this.
    • # Example Scope Configuration for a Policy:
-   Targets:
    -   Choose "All Computers" initially for broad reach, then refine.
    -   Or choose "Specific Computer Groups" and add your "All Faculty Macs" Smart Group.
-   Limitations: (Optional, e.g., only apply in a specific building, or to specific users).
    -   Network Segments
    -   Users
    -   Buildings
    -   Departments
-   Exclusions: (Optional, e.g., exclude "IT Department Macs" from a general policy).
    -   Computer Groups (Static or Smart)
    -   Individual Computers
    -   Users
    -   Network Segments
    -   Buildings
    -   Departments

12. User-Initiated Enrollment & Automated Device Enrollment (ADE/DEP)

These are the primary methods for getting devices under Jamf Pro management.

  • User-Initiated Enrollment:
    • Process: User browses to a Jamf Pro enrollment URL, authenticates (if required), and installs a profile/agent.
    • Use Case: BYOD (Bring Your Own Device), small deployments, devices not purchased through Apple's programs.
  • Automated Device Enrollment (ADE/DEP):
    • Process: Devices are pre-assigned to Jamf Pro in Apple Business/School Manager. When a user first powers on the device and connects to the internet, it automatically contacts Jamf Pro for enrollment instructions.
    • Use Case: Corporate-owned devices, bulk deployments, zero-touch provisioning.
    • Benefits: Enforces supervision, non-removable management profile, streamlined user experience.

13. Security and Compliance

Jamf Pro helps enforce security best practices and meet compliance requirements.

  • Disk Encryption (FileVault): Deploy policies to enforce FileVault encryption on macOS devices. Manage recovery keys.
  • Passcode/Password Policies: Enforce strong passcodes on iOS/iPadOS and complex passwords on macOS.
  • Restrictions: Use Configuration Profiles to restrict device functionality (e.g., disable camera, prevent app installation from unknown sources, disable AirDrop).
  • Gatekeeper & XProtect: Jamf Pro integrates with macOS built-in security features.
  • Security Updates: Manage macOS and app updates to patch vulnerabilities.
  • Compliance Reporting: Generate reports to demonstrate compliance with internal policies or external regulations (e.g., "All devices are encrypted," "All Macs have AV installed").
  • Remote Commands: Issue remote lock or wipe commands for lost/stolen devices. 
    # Example Remote Commands (from a device's inventory record or group action):
-   Lock Device
-   Wipe Device
-   Clear Passcode
-   Restart Device
-   Update Inventory

14. Troubleshooting & Best Practices

A. Troubleshooting:

  • Check Device Connectivity: Ensure the device can reach the Jamf Pro server.
  • Verify APNs: Crucial for real-time commands. Check APNs certificate status in Jamf Pro.
  • Review Logs:
    • Jamf Pro Server Logs: For server-side issues.
    • Device Logs: On macOS, check `/Library/Application Support/Jamf/Jamf.log` for agent activity.
    • `jamf.log` on macOS: 
      tail -f /Library/Application\ Support/Jamf/Jamf.log
    • Console.app (macOS): For system-wide logs related to profiles, software installation.
  • Test Scoping: Double-check that policies/profiles are scoped correctly to the target devices/users and that no exclusions apply.
  • `sudo jamf policy` (macOS): Manually trigger policy execution on a Mac. 
    sudo jamf policy # Runs all applicable policies
sudo jamf policy -event <custom_trigger> # Triggers a specific custom policy
sudo jamf policy -id <policy_id> # Runs a policy by its ID
  • `sudo jamf recon` (macOS): Force an inventory update from a Mac.
  • Check Jamf Pro Server Health: Monitor server resources, database health.

B. Best Practices:

  • Plan Your Deployment: Define your organizational structure, user groups, and device types before configuring Jamf Pro.
  • Use Smart Groups Extensively: Leverage dynamic groups for automation and scalability.
  • Modular Policies & Profiles: Break down complex tasks into smaller, focused policies/profiles (e.g., one policy for Chrome install, another for Wi-Fi settings).
  • Test Thoroughly: Always test new policies, profiles, or packages on a small group of test devices before wide deployment.
  • Leverage Self Service: Empower users and reduce IT workload.
  • Automate as Much as Possible: Use triggers, execution frequencies, and smart groups to minimize manual intervention.
  • Regularly Update Jamf Pro: Keep your Jamf Pro server and Jamf Agent up-to-date for new features, bug fixes, and security patches.
  • Maintain Clean Inventory: Periodically review and remove stale or unmanaged devices.
  • Document Everything: Keep clear records of your configurations, policies, and workflows.
  • Utilize Community Resources: Jamf Nation (community.jamf.com) is an invaluable resource for support, scripts, and best practices.
